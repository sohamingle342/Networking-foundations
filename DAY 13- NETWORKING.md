## Day 17: Cloud Foundations - Introduction to Cloud Computing & Virtualization

### 🧠 Core Concepts Learned
* **What is Cloud Computing Really?** At its core, the cloud isn't magic—it's just "someone else's computer" configured to run applications, databases, and services over the internet instead of inside your own local server room.
* **The Evolution of Cloud Architecture:**
  * **Web Applications & Application Servers:** Moving from local desktop software to browser-accessible web applications backed by database clusters (e.g., MySQL or SQL Server) that split work across multiple nodes for reliability.
  * **Clustering & Replication:** Grouping multiple physical or virtual servers together. If one server dies, the cluster keeps running without losing user data or dropping active sessions.
  * **Terminal Services & Thin Clients:** Centrally processing applications on a remote server while low-cost end-user devices (thin clients) simply display the interface window.
  * **Virtualization & Hypervisors:** Software layers that sit on top of physical hardware to run dozens of isolated virtual operating systems on a single physical machine.
* **Public vs. Private Clouds:**
  * **Public Cloud:** Infrastructure owned and managed by third-party providers (AWS, Azure, GCP) shared across multiple customers.
  * **Private Cloud:** Cloud architecture built exclusively for a single organization, either in their own data center or isolated on dedicated hardware.
